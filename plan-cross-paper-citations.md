## 2. Posting-order plan

The dependency is now **mutual** (each paper cites the other), which changes
the arXiv strategy.

### Option A (recommended) — Stagger, accept a v2 update

1. Post SM-CHMM-AR-Paper first with `alswaidan2026chmm` as
   `arXiv:XXXX.XXXXX` placeholder (acceptable; the cite still renders as
   "(Alswaidan et al., 2026)" in text).
2. Post CHMM-paper second with the real SM arXiv ID filled in for
   `alswaidan2026smchmm`.
3. Submit v2 of SM-CHMM-AR-Paper shortly after with the real CHMM arXiv ID
   filled in for `alswaidan2026chmm`. arXiv v2 is routine.

### Option B — Simultaneous / near-simultaneous submission

Submit both on the same day. arXiv assigns IDs at announcement time
(next business day); as long as both are in the same announcement batch,
you can coordinate the cross-IDs by asking the arXiv moderators or by a
same-day v1 update. This is more coordination, no v2 needed.

### Option C — Drop the reciprocal cite

If the Laplace M-step is treated as "folklore" (weighted-median is textbook
Laplace MLE, embedding in HMM γ-weights is analogous to Liu/Peel's embedding
for Student-t), the SM paper could cite `rabiner1989tutorial, liu1995ml` for
the "weighted-MLE-in-Baum-Welch" recipe and not cite CHMM-paper at all. This
is intellectually weaker because CHMM-paper does claim this combination as
novel, but it eliminates the cycle.

**Recommendation: Option A.** Clean, standard practice, and the v2 update
is a single bib-entry edit.
