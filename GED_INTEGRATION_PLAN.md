# CHMM-GED Integration Plan

Plan for promoting CHMM-GED (per-state Generalized Error Distribution emissions) from the exploratory `run_ged_emissions_ablation.jl` ablation to a fourth headline emission family in the paper, parallel to CHMM-N, CHMM-t, CHMM-L.

Implementation already in place (CHMM-Model: `src/Types.jl`, `src/Compute.jl::baum_welch_ged`, `src/Factory.jl`, `run_ged_emissions_ablation.jl`, `run_ged_robustness.jl`). Validated multiseed (10 seeds, deterministic 13-Gaussian / 4-Laplace partition) and cross-ticker (6/6 tickers, ≥1 heavy-shape state). LL beats CHMM-t at the same parameter count.

This plan inventories every paper artifact that needs to gain a GED row/column/panel, lists the corresponding CHMM-Model script changes, defines the new GED-specific artifacts (the "shape histogram" and bracket sensitivity), and orders the work into phases by criticality.

### 2.3 Defaults to lock in `src/Compute.jl` and `src/Factory.jl`

Based on the headline ablation:

- **`p_bounds = (0.5, 3.0)` as the default.** Tightening `p_max` from 4.0 to 3.0 costs about 4 nats of LL but produces a deterministic partition across 10 seeds and removes wasted ECM iterations on bulk states pinned to `p_max`.
- **`p_init = 1.5`** (between Gaussian and Laplace) as default.
- **`max_iter = 60`** to match the existing CHMM-t default.

These defaults already match `baum_welch_ged`'s current signature except for `p_bounds`, which currently defaults to `(0.5, 4.0)`. Change to `(0.5, 3.0)` as the documented paper-canonical default; runs that need the wider bracket can pass `p_bounds=(0.5, 4.0)` explicitly.

## 4. Risks and decisions

### 4.1 Decisions to make before starting

1. **Cross-ticker main-text table (`tab:cross_ticker`):** keep CHMM-t main text + add CHMM-GED appendix twin (recommended), or replace CHMM-t with CHMM-GED in main text? Keeping CHMM-t avoids re-anchoring the existing narrative; the CHMM-GED twin then becomes the "best LL alternative" appendix entry.
2. **Architecture figure (`fig:chmm_architecture`):** 4-box single row, or 2x2 layout? Single row is tighter but cramped; 2x2 reads cleaner. Recommend 2x2.
3. **SM-CHMM-GED:** include in `tab:extended_baselines` as a 4th SM-CHMM row, or defer? Recommend defer; the SM-CHMM panel exists to show the geometric-sojourn restriction is not the bottleneck, and one row per family is sufficient to make that point.
4. **Price-sim figures (`Fig-{TICKER}-PriceFan-N.pdf` etc.):** generate `-GED` figures, or keep CHMM-N only? The existing appendix uses CHMM-N only by design; the table covers the GED numerics. Recommend keep CHMM-N only.

### 4.2 Implementation risks

- **Algorithm 1 readability.** Adding a 4th family branch makes the unified switch statement long. Mitigation: keep each branch to 2-3 lines; the existing CHMM-L branch is the model.
- **Multi-emission appendix figure clutter.** With 4 families × multiple metrics, the K=18 figure suite is going to be visually heavy. Mitigation: 2x2 sub-figure layouts; keep the per-family full-width comparisons (IS / OoS) at one figure each, not all four together.
- **TikZ schematic.** The `fig:chmm_architecture` figure is hand-crafted TikZ. Adding a 4th box requires a careful pass to maintain spacing; easier to move the three emission boxes into a 2x2 grid.
- **`tab:sensitivity_multi` width.** Already `\resizebox{\textwidth}{!}{...}`. Adding a 7-row CHMM-GED block extends vertically only; no width concern.
- **Bracket sensitivity headline.** Phase 3's `tab:p_bracket` should confirm the bimodal partition is robust to bracket choice (it should, given the multiseed deterministic result). If a sensitivity row shows a divergent partition, that becomes a finding to discuss in `discussion.tex` rather than a bug.

### 4.3 What could fail

- If at K=18 with the tighter `p_bounds=(0.5, 3.0)` default the LL gain over CHMM-t shrinks to within numerical noise (currently ~13 nats with `p_max=4.0`, ~9 nats with `p_max=3.0` based on the headline run), the "best LL at same parameter count" claim weakens. Mitigation: report both bracket choices in the algorithms appendix and let the reader judge; the bimodality finding does not depend on which bracket is used.
- If on some ticker (likely JPM, given the super-Laplace concentration in the cross-ticker run) the CHMM-GED kurtosis overshoots the observed value worse than CHMM-t, the variant choice table needs to be careful about claiming GED as a uniformly-better variant. Mitigation: flag this in the discussion as a JPM-specific super-Laplace artifact, parallel to the CHMM-t lower-bracket discussion already in the paper.
